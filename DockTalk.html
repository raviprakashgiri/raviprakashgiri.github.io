Introduction

DocTalk:



  • A system that extends doctors’ visits by allowing them to share pre-recorded audio messages, in their own voice, to patients with low-end mobile phones.



  • Uses a combination of SMS and interactive voice response (IVR) to orchestrate this communication between doctors and patients.

    • Idea: Awaaz.De

  • It's a web service that allow user to broadcast audio messages that can be received on the mobile phones of the people of his interest.


  • User can upload audio on the web or can record there voice by calling to a no. and that can be broadcasted to the mobile phones's of each one he wants.


  • Each call will be following a message that contain a no. on which that one can call back to listen that message again.

    • Overview

    What is the Need of this system?



  • Uncommon for doctors to see 100 patients each day. ( specially in rural areas)


  • Time constraints for detailed instructions.


  • Unavailability of electronic resources to the rural people like internet etc.

    • Our Goals:



  • Our goal is to explore whether mobile phones can be used to amplify the eﬀectiveness of doctors’ time in over-crowded clinics.


  • Instead of doing a live counseling session for each patient, can we record a single counseling session – recorded once in compassionate and meticulous detail – and make the audio available for all subsequent patients?.

    • How is it different?



  • DockTalk is unique in that as it preserve the personal relationship between doctors and patients.


  • Patients are prescribed a message during there a face to face meeting with the doctor that they can listen on there phone.

    • Usage Scenario

    Doctor want to record message for fever patients.



    Step 1: Sending request string " Record Fever " to the DockTalk.


    Step 2: System respond by calling back to record with tag Fever.


    Step 3: For sharing, doctor send message " Share (tag) (phone no) " sms read to the DockTalk with patient mob no. and tag.

    • Deployment

    Initial

    Deployment



    DockTalk was first deployed using a variant of IVR Junction, its an open system that runs on Windows.


    It was using Voxeo Prophecy as the IVR engine and Topex Mobilink gsm modem.


    It was deployed in Mumbai, Bangalore and some other places and was tested with 21 doctors.

    Outcomes



  • Initial feedback was good, no. of doctors took interest and adopted.But they didn't want to be the first user.

  • Due to there demanding schedule, doctors were hesitant to learn the system and record the message.

  • No positive feedback came until they were sure that it'll give value to them.

    • Conclusion



    • DockTalk IVR system is simple to use, no navigation or button press required.


    • The system could also benefit pharmacists, as they can prescribe instructions for taking medications.


    • As a future implementation, a smart phone application might make it easier for doctors to record and prescribe messages.
